# 과제(9) realloc을 이용한 순차 리스트 개선 보고서

학과:컴퓨터공학과 23학번  
이름:김주혁  

## 1. 과제 개요
배열 기반 순차 리스트의 한계인 포화 상태(Full)를 해결하기 위해, `realloc` 함수를 활용하여 데이터 추가 시 자동으로 배열 용량을 확장하는 프로그램을 작성하였습니다.

## 2. 코드 구현 및 교수님 힌트 분석
- **포화 상태 해결:** 데이터 삽입 전 size == capacity조건을 검사하고, 포화 상태일 경우 realloc을 호출하여 배열 용량을 2배로 늘린 후 기존 데이터를 안전하게 이전시켰습니다.
- **오류 방지 (힌트 분석):** 교수님께서 언급하신 할당 부분의 실수힌트를 바탕으로 실습 코드를 점검했습니다. 
  - 리스트의 데이터 타입(예: int* array)과 동적 할당 시 곱해지는 크기(sizeof(int))가 정확히 일치해야 함을 확인했습니다. 
  - 만약 포인터는 8바이트 데이터(예: double 또는 큰 구조체)를 가리키는데, realloc에서는 sizeof(int)(4바이트)를 사용해버리면 치명적인 메모리 침범 오류가 발생합니다. 
  - 본 과제에서는 int *temp = (int *)realloc(..., new_capacity * sizeof(int)); 처럼 자료형과 sizeof 연산을 완벽하게 일치시켜 이러한 논리적 오류를 원천 차단했습니다.

## 3. 실행 결과
초기 용량을 2로 설정하고 데이터를 연속으로 입력한 결과, 3번째 데이터 입력 시 용량이 4로, 5번째 데이터 입력 시 용량이 8로 안정적으로 두 배씩 확장되며 정상 작동함을 확인했습니다.
